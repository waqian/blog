我是个退休闲人，50多岁，现在每天最大的烦恼是：闲。

从没正经写过一行代码。但这不妨碍我想搞个"持仓季度风险评价系统"。

没钱？没关系。没技术？也不是事。

我的武器库：免费的 Claude、Codex，加上腾讯云 Coding Plan。

今天就跟大家唠唠，一个纯菜鸟是怎么靠"使唤傻儿子"把系统搞出来的。

---

## 一、需求分析：和 AI 聊天要带脑子

开发的第一步不是写代码，而是搞清楚你到底想干什么。

### 我的套路：

**1. 资料提前准备好**

把想法整理成 `.md` 文件，别让 AI（Claude)  跟你玩猜谜。

这就像去餐厅点菜——别光说"我要吃好吃的"，把菜单研究清楚再开口。

**2. 多轮问答，发现不对直接改问题**

想到哪聊到哪，发现漏洞就往回改。

别怕"打断"对话——AI 不会嫌你烦，token 烧的是服务商的钱（免费额度内）。

**3. 聊完总结成文档**

让 AI 帮你整理一份正式的需求文档。

这份文档就是后面所有开发的"圣经"，写错了后面全白干。

### 省额度的秘诀：

每一轮对话都烧 token。把资料准备好，避免重复说车轱辘话。

免费额度就像退休金——省着花，能撑更久。

---

## 二、系统架构：把大象装冰箱分几步？

有了需求文档，下一步是规划"怎么做"。

### 关键技巧：

**1. 先发文档，明确说"别写代码"**

Claude 有个毛病，上来就甩你一堆脚本。

你得按住它的手："先别动手，先动脑子。"

**2. 把需求拆成小步骤**

每个步骤尽量独立，小单元应对免费额度限制。

这就像切西瓜——一口吃不下，切成片慢慢嚼。

**3. 多轮讨论，把技术细节敲定**

每一步怎么实现、用什么方案、边界条件是什么。

聊透了再动工，比边写边改省十倍精力。

**4. 写测试用例和验收方案**

别等代码写完了才发现做错了。

测试用例就是你的"验货清单"，菜不可怕，可怕的是菜还不知道自己菜。

## 三、代码开发：学霸要用来找错，不是用来写作业

终于开始写代码了。

但怎么用 AI 才最聪明？

我琢磨出一套"傻儿子开发法"。

### 核心原则：

> **让学渣干粗活，学霸找错误。**

学霸做简单题太浪费，让他们改卷子才是人尽其才。

### Layer 1：第一步开发

**1. 找"傻儿子"（lkeap）写初稿**

用lkeap ，便宜、能跑、肯干活。

让它写第一遍，费 token 是费了点，但能把框架搭起来。

**2. 让学霸检查——Claude Code / Sonnet**

学霸的强项：找 bug、看逻辑漏洞、记性好。

你让它从头写，它嫌累；你让它改别人的代码，它来劲。

**3. 让 Codex 跑测试**

免费的 Codex 专门跑测试用例。

测完有错？回到步骤2，让学霸继续找。

### Layer 2：第二步开发

**1. 再找"傻儿子"写第二块功能**

还是一样的套路，最小化开发目标。

**2. 学霸检查 Layer 2**

**3. Codex 测试 Layer 1 + Layer 2 的集成**

这一步很重要——两块代码单独能跑，拼在一起可能打架。

### 如此往复：

Layer 3、Layer 4……

直到项目全部完成。

### 为什么这样分工？

| 角色                  | 干什么 | 为什么                       |
| --------------------- | ------ | ---------------------------- |
| 傻儿子（lkeap）       | 写初稿 | 便宜、量大、不嫌累           |
| 学霸（Claude/Sonnet） | 找错误 | 眼尖、记性好、改别人代码来劲 |
| Codex                 | 跑测试 | 免费、客观、不讲情面         |

**分工明确，效率拉满。**
